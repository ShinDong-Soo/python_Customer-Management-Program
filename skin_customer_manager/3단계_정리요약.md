# 피부관리샵 고객 관리 프로그램 — 3단계 정리 요약 (완료)

## 1. 3단계 목표

2단계 **CSV 기반 고객·방문 관리**를 **SQLite 데이터베이스**로 전환하고, SQL CRUD·검색·JOIN 조회를 적용했습니다.

```text
2단계: customers.csv + visits.csv (파일 저장)
   ↓
3단계: skin_shop.db (SQLite 테이블 + Foreign Key)
```

- **언어**: Python (표준 라이브러리 `sqlite3`)
- **저장 방식**: SQLite (`skin_shop.db`)
- **현재 커밋**: `f84ff86` — Migrate to SQLite with search, JOIN queries, and stage 3 tests

## 3. 아키텍처 변경

### 역할 분리

| 파일 | 역할 |
|------|------|
| `main.py` | 사용자 입력, 메뉴, 검증, 출력 |
| `database.py` | DB 연결, SQL 실행, 데이터 변환 |

2단계에서는 `main.py` 안에 CSV 읽기/쓰기 로직이 함께 있었지만, 3단계에서는 **데이터 접근 계층**을 `database.py`로 분리했습니다.

### DB 초기화 흐름

```text
main() 시작
  → initialize_database()
      → get_connection()
      → create_customers_table()
      → create_visits_table()
      → commit / close
```

## 4. 데이터베이스 구조

### customers 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 고객 고유 번호 |
| name | TEXT NOT NULL | 이름 |
| phone | TEXT NOT NULL | 연락처 |
| birth | TEXT | 생년월일 |
| skin_type | TEXT | 피부 타입 |
| memo | TEXT | 메모 |
| created_at | TEXT | 등록일 |

### visits 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 방문 기록 고유 번호 |
| customer_id | INTEGER NOT NULL | 연결된 고객 ID (FK) |
| date | TEXT NOT NULL | 방문 날짜 (YYYY-MM-DD) |
| service | TEXT NOT NULL | 시술 내용 |
| price | INTEGER NOT NULL | 가격 |
| memo | TEXT | 메모 |

### Foreign Key

```sql
FOREIGN KEY (customer_id) REFERENCES customers(id)
```

- `PRAGMA foreign_keys = ON`으로 FK 제약을 활성화
- 방문 기록이 남아 있는 고객은 DB 레벨에서도 삭제가 차단됨 (`IntegrityError`)


## 6. database.py 주요 함수

### 연결·초기화

| 함수 | 역할 |
|------|------|
| `get_connection()` | DB 연결, Row 팩토리 설정, FK 활성화 |
| `initialize_database()` | 테이블 생성 |
| `create_customers_table()` | customers 테이블 생성 |
| `create_visits_table()` | visits 테이블 + FK 생성 |

### 고객 CRUD

| 함수 | SQL | 역할 |
|------|-----|------|
| `load_customers()` | SELECT * | 전체 고객 조회 |
| `get_customer_by_id()` | SELECT … WHERE id = ? | ID로 고객 1명 조회 |
| `insert_customer()` | INSERT | 고객 등록 |
| `update_customer()` | UPDATE | 고객 수정 |
| `delete_customer()` | DELETE | 고객 삭제 |
| `search_customers()` | SELECT … WHERE name/phone LIKE ? | 이름·연락처 검색 |

### 방문 CRUD / 조회

| 함수 | SQL | 역할 |
|------|-----|------|
| `load_visits()` | SELECT * | 전체 방문 조회 |
| `get_visits_by_customer_id()` | SELECT … WHERE customer_id = ? | 고객별 방문 |
| `insert_visit()` | INSERT | 방문 등록 |
| `customer_has_visits()` | SELECT 1 … LIMIT 1 | 방문 존재 여부 |

### JOIN 조회

| 함수 | 역할 |
|------|------|
| `get_visits_with_customer_by_id()` | 특정 고객의 방문 + 고객명 JOIN |
| `get_all_visits_with_customer()` | 전체 방문 + 고객명 JOIN |

### 변환 헬퍼

| 함수 | 역할 |
|------|------|
| `_row_to_customer()` | sqlite3.Row → 고객 dict |
| `_row_to_visit()` | sqlite3.Row → 방문 dict |
| `_row_to_visit_join()` | JOIN 결과 Row → 방문+고객명 dict |

## 7. 2단계 → 3단계 주요 변경

| 항목 | 2단계 (CSV) | 3단계 (SQLite) |
|------|-------------|----------------|
| 저장 | `customers.csv`, `visits.csv` | `skin_shop.db` |
| ID 생성 | `get_next_id()` 수동 계산 | `AUTOINCREMENT` |
| 검색 | Python 리스트 필터링 | `LIKE` SQL 검색 |
| 방문 조회 | Python에서 customer_id 필터 | SQL WHERE / JOIN |
| 관계 보장 | 코드로만 체크 | Foreign Key + 코드 검증 |
| 데이터 접근 | main.py 내부 함수 | database.py 분리 |

## 8. SQL 예시

### 검색 (메뉴 3)

```sql
SELECT * FROM customers
WHERE name LIKE ? OR phone LIKE ?
ORDER BY id
```

### 고객별 JOIN 조회 (메뉴 7)

```sql
SELECT
    visits.id AS visit_id,
    visits.customer_id,
    visits.date,
    visits.service,
    visits.price,
    visits.memo,
    customers.name AS customer_name
FROM visits
INNER JOIN customers ON visits.customer_id = customers.id
WHERE customers.id = ?
ORDER BY visits.id
```

### 전체 JOIN 조회 (메뉴 8)

```sql
SELECT … (동일)
FROM visits
INNER JOIN customers ON visits.customer_id = customers.id
ORDER BY visits.id
```

## 13. 블로그 정리 포인트

> 3단계에서는 CSV 파일 저장 방식을 SQLite로 전환하고,  
> **데이터 접근 로직을 `database.py`로 분리**했다.
