## 3. 4.5-A — 실사용 강화

### 3-1. 연락처 중복 방지

| 항목 | 내용 |
|------|------|
| 함수 | `get_customer_by_phone()` (`database.py`) |
| 검사 | `prepare_customer_data()` (`services.py`) |
| 정책 | 동일 연락처 등록 차단 (DB UNIQUE 대신 앱 검사) |
| 수정 시 | 본인 ID 제외 후 중복 검사 |
| 안내 | `이미 등록된 연락처입니다. 기존 고객: 이름 (ID: N)` |

### 3-2. DB 백업·복원

| 기능 | 설명 |
|------|------|
| **DB 백업** | `skin_shop.db` → `backup/skin_shop_YYYY-MM-DD_HHMMSS.db` |
| **백업 복원** | 백업 목록에서 선택 → 현재 DB 덮어쓰기 |
| GUI 위치 | 창 상단 `데이터 관리` — [DB 백업] [백업 복원] |

```text
backup/
 └─ skin_shop_2026-07-12_230500.db
```

### 3-3. services.py — 로직 분리

| 함수 | 역할 |
|------|------|
| `prepare_customer_data()` | 고객 입력 검증 + 연락처 중복 |
| `prepare_visit_data()` | 방문 입력 검증 |
| `check_customer_deletable()` | 방문 있는 고객 삭제 차단 |
| `check_customer_updatable()` | 수정 대상 존재 확인 |
| `check_visit_deletable()` | 방문 삭제 대상 확인 |
| `check_visit_updatable()` | 방문 수정 대상 확인 |

```text
gui.py  →  services.py  →  database.py
main.py →  services.py  →  database.py
```

## 4. 4.5-B — 목록·정렬 강화

### 4-1. 고객 목록 통계 컬럼

| 컬럼 | SQL | 설명 |
|------|-----|------|
| 최근 방문일 | `MAX(v.date)` | 마지막 방문 날짜 |
| 방문 횟수 | `COUNT(v.id)` | 누적 방문 수 |
| 누적 금액 | `COALESCE(SUM(v.price), 0)` | 시술 금액 합계 |

**함수:** `load_customers_with_stats()`, `search_customers_with_stats()`

```sql
SELECT c.*, MAX(v.date), COUNT(v.id), SUM(v.price)
FROM customers c
LEFT JOIN visits v ON c.id = v.customer_id
GROUP BY c.id
```

### 4-2. Treeview 정렬

| 항목 | 내용 |
|------|------|
| 방법 | 컬럼 헤더 클릭 |
| 표시 | 정렬 중인 컬럼에 ▲ / ▼ |
| 대상 | 고객 목록, 방문 내역 모두 |
| 기본값 | 방문 목록 — **방문 날짜 최신순** |

### 4-3. 방문 변경 시 연동

방문 **등록 / 수정 / 삭제** 후 고객 탭 통계(최근 방문일·횟수·금액) 자동 새로고침.

## 5. 버그 수정

### load_customers NameError

| 항목 | 내용 |
|------|------|
| 증상 | `python gui.py` 실행 시 프로그램 시작 직후 오류 |
| 원인 | 4.5-B에서 `load_customers` import 제거 후 `_refresh_visit_customer_combo()`에서 미수정 호출 |
| 수정 | `self.current_customers or load_customers_with_stats()` 사용 |

## 10. 블로그 정리 포인트

> 4단계 GUI 이후, 연락처 중복 방지·DB 백업으로 **실매장 운영**을 고려했고,

> **단순 GUI가 아닌 로컬 CRM** 수준으로 보완했다.
