# 피부관리샵 고객 관리 프로그램 — 1단계 정리 요약

## 1. 프로젝트 개요

콘솔(터미널) 기반으로 고객 정보를 관리하고, CSV 파일에 저장하는 프로그램입니다.

- **언어**: Python
- **저장 방식**: CSV (`customers.csv`)
- **현재 커밋**: `fabb7fb` — Improve customer manager validation and UX

---

## 2. 폴더 구조

```text
skin_customer_manager/
 ├─ main.py              # 메인 프로그램
 ├─ customers.csv        # 고객 데이터 (로컬, git 제외)
 └─ 1단계_정리요약.md    # 이 문서
```

---

## 3. 고객 데이터 항목

| 항목 | 설명 |
|------|------|
| id | 고객 고유 번호 (자동 부여) |
| name | 이름 |
| phone | 연락처 |
| birth | 생년월일 |
| skin_type | 피부 타입 |
| memo | 메모 |

---

## 4. 구현 기능

| 메뉴 | 기능 | 설명 |
|------|------|------|
| 1 | 고객 등록 | 새 고객 정보 입력 후 CSV 저장 |
| 2 | 고객 목록 조회 | 전체 고객 목록 출력 |
| 3 | 고객 검색 | 이름 또는 연락처로 검색 |
| 4 | 고객 정보 수정 | ID 기준으로 항목별 수정 |
| 5 | 고객 삭제 | ID 기준으로 고객 삭제 |
| 0 | 종료 | 프로그램 종료 |

---

## 5. 주요 함수 구조

| 함수 | 역할 |
|------|------|
| `initialize_csv()` | CSV 파일 없으면 헤더만 생성 |
| `load_customers()` | CSV에서 고객 목록 읽기 |
| `save_customers()` | 고객 목록을 CSV에 저장 |
| `get_next_id()` | 다음 고객 ID 자동 생성 |
| `add_customer()` | 고객 등록 |
| `show_customers()` | 고객 목록 조회 |
| `search_customer()` | 고객 검색 |
| `update_customer()` | 고객 정보 수정 |
| `delete_customer()` | 고객 삭제 |
| `print_customer()` | 고객 정보 출력 (공통) |
| `show_menu()` | 메뉴 출력 |
| `main()` | 프로그램 실행 진입점 |

---

## 6. 추가 반영된 개선 사항

1단계 기본 CRUD 이후 아래 기능이 추가되었습니다.

- **출력 공통화**: `print_customer()` 함수로 목록/검색 출력 통일
- **총 고객 수 표시**: 목록 조회 시 `총 고객 수 : N명` 출력
- **필수 입력 검증**: 이름, 연락처 공백 입력 방지
- **삭제 확인**: 삭제 전 `Y/N` 확인 후 진행

---

## 7. 실행 방법

```bash
cd skin_customer_manager
python main.py
```

> `main.py`는 `skin_customer_manager` 폴더 안에 있습니다.  
> 저장소 루트에서 `python main.py`를 실행하면 파일을 찾지 못합니다.

---

## 8. CSV 파일 동작

- 프로그램 시작 시 `customers.csv`가 없으면 자동 생성됩니다.
- 고객 등록/수정/삭제 시마다 CSV 파일이 갱신됩니다.
- `customers.csv`는 `.gitignore`에 등록되어 원격 저장소에 올라가지 않습니다.
