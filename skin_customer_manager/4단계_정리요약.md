# 피부관리샵 고객 관리 프로그램 — 4단계 정리 요약 (완료)

## 1. 4단계 목표

3단계 **콘솔 + SQLite** 프로그램을 **Tkinter GUI**로 전환하여, 매장에서 마우스로 조작할 수 있는 화면을 만들었습니다.

```text
3단계: main.py (콘솔)  →  database.py  →  skin_shop.db
   ↓
4단계: gui.py (Tkinter) →  database.py  →  skin_shop.db
```

- **UI**: Tkinter (Python 표준 라이브러리)
- **DB**: SQLite (`skin_shop.db`) — 3단계와 동일
- **실행**: `python gui.py`

## 10. 테스트

### 자동 테스트 (DB)

```bash
cd skin_customer_manager
python test_stage3.py
```

### 수동 테스트 (GUI)

```text
1. python gui.py 실행
2. [고객 관리] 고객 등록 → 목록 표시 확인
3. 목록 클릭 → 수정 → 반영 확인
4. 검색 → 결과 확인 → 전체 보기
5. [방문 기록] 고객 선택 → 방문 등록 → 목록 표시
6. 방문 선택 → 수정 / 삭제
7. 방문 있는 고객 삭제 시도 → 차단 확인
8. 프로그램 종료 후 재실행 → 데이터 유지 확인
```

### 참고: ID가 1이 아닌 경우

SQLite `AUTOINCREMENT`는 삭제된 ID를 재사용하지 않습니다.  
이전 테스트에서 ID 1이 쓰였다가 삭제되면, 다음 고객은 ID 2부터 시작합니다. 정상 동작입니다.

## 15. 4.5-B 보완 (목록·정렬)

| 항목 | 내용 |
|------|------|
| 최근 방문일 | 고객 목록 컬럼 (`MAX(v.date)`) |
| 방문 횟수 | `COUNT(v.id)` |
| 누적 금액 | `COALESCE(SUM(v.price), 0)` |
| Treeview 정렬 | 컬럼 헤더 클릭 → 오름/내림 (▲▼) |
| 방문 변경 시 | 고객 통계 자동 새로고침 |

### 정렬 사용법

- 고객·방문 목록 **컬럼 제목 클릭** → 해당 기준 정렬
- 같은 컬럼 다시 클릭 → 오름/내림 전환
- 방문 목록 기본: **방문 날짜 최신순**
